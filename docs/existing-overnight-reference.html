🔍 Scanner Insights

Key Methodology Finding

Broad scanner caught mostly ±120s boundary artifacts. The 120-second correlation window creates edge effects where signals artificially cluster at ±120s boundaries rather than representing real market lag.

Why this happens: When the correlation window equals the maximum lag detection range (120s), edge effects dominate. Real signals need dedicated live WebSocket tests with longer observation windows (300s+) and continuous data streams.

This is a methodology insight, not a failure. The overnight scanner successfully validated that:

  • Broad scanning requires longer correlation windows
  • Live testing beats historical analysis for lag detection
  • Exchange discovery (9 exchanges found) was successful
  • Previous research findings (WIF 28s lag) remain valid
